# two-player.app

> A premium, mobile-to-TV gaming ecosystem designed for couples to play high-stakes, interactive games together. The platform serves as a central hub for polished, web-based gaming experiences optimized for casting from mobile devices to a TV.

## 🎯 Quick Start / Core Entrypoints
- Primary Entrypoint: `index.html` - The central landing page and game hub.
- Game Entrypoints: 
    - `dirty_guesser_web/index.html` - The entrypoint for "Dirty Guesser", a game of deduction and deception.
    - `neonfeud/index.html` - The entrypoint for "Neon Feud", the gold standard for design and interactivity.
- Development Command: `n/a` (Static site; use a local live server like `npx serve` or VS Code Live Server)
- Production/Build Command: `n/a` (Deployed via GitHub Pages)

## 🏗️ Core Architecture & Directory Map
- `[root/]` - Contains the main landing page (`index.html`), global styles (`landing.css`), and game modules.
- `[dirty_guesser_web/]` - Dedicated module for the Dirty Guesser game.
    - `dirty_guesser_web/js/` - Game logic for Dirty Guesser.
    - `dirty_guesser_web/css/` - Game-specific styles.
    - `dirty_guesser_web/data/` - Scenario data (`scenarios.json`).
    - `dirty_guesser_web/audio/` - Game-specific sound effects.
- `[neonfeud/]` - Dedicated module for the Neon Feud game (The Gold Standard).
    - `neonfeud/js/` - Game logic (`script.js`).
    - `neonfeud/css/` - Game-specific styles (`styles.css`).
    - `neonfeud/audio/` - Game-specific sound effects.
- Configuration Files: Located in `neonfeud/manifest.json` and `neonfeud/packs.json`.

## 🛠️ Tech Stack & Key Dependencies
- Language/Runtime: `HTML5`, `CSS3`, `Vanilla JavaScript`
- Primary Frameworks: None (High-performance vanilla implementation)
- Deployment: `GitHub Pages`

## ⚠️ Anti-Patterns, Constraints & Guardrails
- **AESTHETIC STANDARD:** All new game modules or UI elements must match the high-end, polished "Neon Feud" aesthetic. Do not settle for basic or "placeholder" looking components.
- **UX CONSTRAINT:** Design must prioritize mobile-first interaction (for the controller/phone) with TV-optimized visual output (for the display).
- **CODE STYLE:** Maintain strict modularity; keep game logic isolated within its respective directory. Avoid global namespace pollution.
- **LOCAL PERFORMANCE DIRECTIVE:** When providing code updates, do not use placeholders like `// ... rest of code`. Provide complete, functional blocks to ensure seamless integration into the existing gaming ecosystem.
